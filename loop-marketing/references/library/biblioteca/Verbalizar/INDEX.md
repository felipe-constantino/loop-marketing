# Biblioteca Tática — Verbalizar (fase Express)

> Recursos táticos do **verbalizar-agent**. NÃO são comandos invocáveis pelo usuário — são arquivos de aprofundamento que o agente lê **sob demanda** quando precisa produzir um entregável específico de mensagem/posicionamento.
>
> **Como usar (instrução para o agente):**
> 1. Identifique a necessidade tática durante o método base.
> 2. Escolha **UM** arquivo abaixo pelo "use quando".
> 3. Leia `biblioteca/Verbalizar/<arquivo>.md` e use as instruções dele como **insumo**.
> 4. **O resultado volta SEMPRE pelo contrato de saída do verbalizar-agent**: gating de maturidade, teste de diferenciação, teste de especificidade, grau de confiança e handoff. O prompt da biblioteca gera matéria-prima; a decisão continua sendo do agente.
>
> Regra: no máximo 1-2 arquivos por ciclo. Se nenhum "use quando" casa, **não force** — siga o método base do agente.

## Índice

- **otimizador-de-proposta-de-valor.md** — use quando: precisa criar ou refinar a proposta de valor central (L1).
- **construtor-de-hierarquia-de-mensagens.md** — use quando: precisa estruturar a hierarquia L1 → L2 → L3 com mensagens de suporte e provas.
- **decodificador-de-idioma-do-cliente.md** — use quando: precisa extrair a linguagem real do cliente e montar guia de tom de voz (Passo 1 do agente).
- **voz-do-sintetizador-de-cliente.md** — use quando: tem transcrições/conversas de clientes e precisa sintetizá-las em insights de mensagem.
- **analisador-de-diferenciacao-de-marcas.md** — use quando: a mensagem falhou no teste de diferenciação e precisa de posicionamento distinto.
- **analise-de-voz-competitiva.md** — use quando: precisa mapear como os concorrentes se comunicam para achar espaço de voz.
- **localizador-de-lacunas-do-concorrente.md** — use quando: busca lacunas de posicionamento que os concorrentes deixam aberto.
- **perfil-do-cliente-ideal.md** — use quando: o ICP está indefinido ou genérico e precisa ser construído a partir de dados.
- **buyer-persona-validador.md** — use quando: já existe persona e precisa validá-la contra dados reais do cliente.
- **mapeador-da-jornada-do-cliente.md** — use quando: precisa mapear emoções, perguntas e pontos de contato por estágio da jornada.
- **analisador-de-padroes-de-sucesso-do-cliente.md** — use quando: quer identificar padrões dos clientes bem-sucedidos para replicar na mensagem.
- **arquiteto-de-historias-de-marcas.md** — use quando: precisa de narrativa/storytelling de marca (consideração/awareness).
- **criador-de-perfil-de-personalidade-da-marca.md** — use quando: precisa definir personalidade/voz consistente da marca.
- **identificador-do-arquetipo-da-marca.md** — use quando: quer ancorar a voz num arquétipo de marca (12 arquétipos).
- **esclarecimento-de-objetivo-e-missao.md** — use quando: o propósito/missão não está claro e contamina a mensagem.
- **definidor-de-categoria-de-mercado.md** — use quando: o problema é a categoria de mercado em que a marca se coloca.
- **posicionador-de-lideranca-de-pensamento.md** — use quando: a estratégia pede autoridade/thought leadership.
- **posicionamento-da-parceria.md** — use quando: precisa posicionar parcerias ou co-branding.
- **analise-de-interruptores-do-setor.md** — use quando: a tese é contrariar práticas do setor (disrupção de mensagem).
- **gerador-de-conceito-de-campanha.md** — use quando: precisa de conceitos criativos de campanha sobre a mensagem definida.
- **gerador-de-tema-de-conteudo.md** — use quando: precisa de temas/eixos de conteúdo derivados da hierarquia de mensagens.
- **auditoria-e-estrategia-de-conteudo.md** — use quando: existe biblioteca de conteúdo a auditar e alinhar à marca/jornada.
- **desenvolvedor-de-estrategia-sazonal.md** — use quando: precisa de calendário/eixo sazonal de mensagem.
- **planejador-de-ativacao-de-marcas.md** — use quando: há lançamento/rebranding e precisa de plano de ativação.
- **roteiro-de-evolucao-das-marcas.md** — use quando: precisa de roadmap de evolução de marca de longo prazo.
